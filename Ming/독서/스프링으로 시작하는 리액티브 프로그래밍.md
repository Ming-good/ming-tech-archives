#### 2025.02.07 Sinks
1. Sinks의 동작방식 : 멀티스레드에서 스레드 세이프하다. 그러므로 여러개의 스레드에서 각각 전혀 다른 작업을 처리 후, 처리 결과를 반환하는 상황이 발생할 수 있는데 이 같은 상황에 적절하게 사용할 수 있는 방식이 Sinks이다.
2. Sinks의 종류
	1. Sinks.One : mit 된 데이터 중에서 단 하나의 데이터만 Subscriber에게 전달한다. 나머지 데이터는 Drop 됨
	2. Sinks.Many
		1. unicast() : UnicastSpec을 리턴하고 사용함. 1:1 방식임. (단 하나의 Subscriber에게 데이터 emit)
		2. muticast() : MulticastSpec을 리턴하고 1:MANY 방식이며 Warm up의 특징이 있다.
		3. replay().limit(int) : emit된 데이터 중에서 특정 시점으로 되돌린 데이터부터 emit
