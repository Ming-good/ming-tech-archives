## 처리율 제한 알고리즘
#### 토큰 버킷 알고리즘
- 각 요청이 처리될 때 마다 하나의 토큰을 사용한다. 
	- 토큰이 있을 경우: 버킷에서 토큰을 하나 꺼낸 후 요청을 처리한다.
	- 토큰이 없는 경우: 해당 요청은 버려진다.
- 토큰은 토큰 공급기(refiler)에 의해 매초 토큰을 추가한다.
- 장점:
	- 구현이 쉽다.
	- 메모리 사용 측면에서도 효율적이다.
	- 짧은 시간에 집중되는 트래픽도 처리 가능하다.
- 단점:
	- 이 알고리즘은 버킷 크기와 토큰 공급률이라는 두 개 인자를 가지고 있는데, 이 값을 적절하게 튜닝하는 것은 까다로운 일 이 될 것이다.
#### 누출 버킷 알고리즘
- 누출 버킷 알고리즘은 토큰 버킷 알고리즘과 비슷하지만 요청 처리율이 고정되어 있다는 점이 다르다.
- 보통 FIFO큐로 구현한다.
- 처리 단계:
	- 요청이 도착하면 큐가 가득 차 있는지 확인하고 빈자리가 있는 경우 큐에 요청을 추가한다.
	- 큐가 가득 차 있는 경우 새 요청은 버린다.
	- 지정된 시간마다 큐에서 요청을 꺼내어 처리한다. 
		- 지정된 시간당 몇 개의 항목을 처리할지 처리율을 지정
- 장점: 
	- 큐의 크기가 제한되어 있어 메모리 사용량 측면에서 효율적
	- 고정된 처리율을 갖고 있기 때문에 안정적 출력이 필요한 경우 적합
- 단점: 
	- 단시간에 많은 트래픽이 몰리는 경우 큐에는 오래된 요청들이 쌓이게된다.
	- 두 개의 인자를 갖고 있는데, 이들을 올바르게 튜닝하기가 까다로울 수 있다.
