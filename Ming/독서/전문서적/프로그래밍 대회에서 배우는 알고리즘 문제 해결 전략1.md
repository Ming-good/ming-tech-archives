## 문제 해결 개관
##### 문제 해결 알고리즘
1. 문제를 읽고 이해한다.
	- 문제를 공격적으로 읽고 이해하는 과정이 필요하다. 사소한 제약조건 하나로 오답을 낼 수 있기 때문
2. 익숙한 용어로 문제를 재정의 한다.
	- 문제의 본질을 어떻게 재구성하느냐에 따라서 같은 일을 하는 프로그램이라도 전혀 다른 동작을 할 수 있다. 
	- 추상화 과정이 프로그래밍이 나아갈 방향을 결정한다.
3. 문제 해결 계획을 세운다
4. 계획을 검증한다
5. 프로그램으로 구현한다.
6. 어떻게 풀었는지 돌아보고, 개선할 방법이 있는지 찾아본다.
	- 문제를 풀 때마다 경험을 기록으로 남기는 것
	- 문제를 틀렸을 경우 오답노트를 할 것
	- 시간 내 해결하지 못 했을 경우 다른 사람들의 코드를 볼 것
#### 문제 해결 전략
1. 문제를 분류하는 방법을 익힐 것
2. 단순한 방법도 생각해보는 것도 나름의 의미가 있다. 단순한 방법은 알고리즘의 효율성의 기준성을 정해주기 때문 (점진적 개선)
3. 문제의 단순화
4. 그림으로 그려볼 수 있을까? (직관 사용)
5. 수식으로 표현하여 사고를 전개하는 접근 방식
6. 문제를 분해할 수 있을까? (더 다루기 쉬운 형태로 변형하는 것)
7. 뒤에서부터 생각해서 문제를 풀 수 있을까?
8. 순서를 강제할 수 있을까?
