## 메서드 참조
1. [[메서드 참조란]]
	- 메서드 참조란 무엇인지
	- 메서드 참조는 브릿지 역할일 때 사용하면 직관적이고 간결한 코드를 작성할 수 있다.
	- 메서드 참조의 장점
		- 간결, 가독
		- 매개변수 명시적 작성 필요가 없음
		- 별도 로직 분리로 재사용성 상승
> **2025.07.06**
2. [[메서드 참조의 4가지 유형]]
	- 정적 메서드 참조: `Class::method`
	- 특정 객체의 인스턴스 메서드 참조: `obj::method`
	- 생성자 참조: `Class::new`
	- 임의 객체의 인스턴스 메서드 참조: `Class::method`
		- `Function<Class, String>`: 매개변수 없는 메서드
		- `BiFunction<Class, Integer, String>`: 매개변수 있는 메서드
	- (특정 VS 임의) 객체의 인스턴스 메서드 참조 차이점
		- 특정: 선언시점 인스턴스 고정
		- 임의: 실행시점 인스턴스 고정
