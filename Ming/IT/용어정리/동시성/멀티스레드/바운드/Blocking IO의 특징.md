## 특징
- 작업이 종료 될 때 가지 요청 스레드가 차단된다.
- 스레드가 차단되는 문제를보완하기 위해 멀티스레딩 기법을 사용할 수 있다.
- 멀티스레딩 기법 사용시 컨텍스트 스위칭 전환 비용, 메모리 사용 오버헤드, 스레드 풀의 응답 지연 등의 문제가 발생할 수 있다.


## 컨텍스트 스위칭 비용 문제
- I/O가 발생할 때 스레드는 대기상태가 되며 I/O작업이 끝낟고 응답 받을 때 해당 스레드는 다시 깨어나게 된다. 
![[Pasted image 20250804001116.png]]

- 이 과정에서 [[컨텍스트 스위칭]]으로 인한 스레드 전환 비용이 발생한다. 
- 즉, CPU는 스레드 전환시 프로세스 상태를 저장, 불러오기를 함으로 전환 비용이 발생하는 것이다.
![[Pasted image 20250804001202.png]]
> 자세히보면 P1이 끝나는 시점에 P2가 바로 실행되는 것이 아니고 어느정도 대기가 존재하는 것을 알 수 있다.
> 이렇게 프로세스가 PCB를 저장하고 reload하는 시간 동안 CPU는 다른 작업을 하지 못 하고 대기하게 된다.
- 컨텍스트 스위칭이 많으면 많을수록 CPU 전체 대기 시간은 길어지고 성능이 저하된다.
