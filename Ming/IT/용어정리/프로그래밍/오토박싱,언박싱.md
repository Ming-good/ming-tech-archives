### ✅ 오토박싱 (Auto-boxing)
- **기본형(primitive type)을 자동으로 래퍼 클래스 객체로 변환**하는 것.
- 예:
```java
int x = 10; 
Integer y = x;  // int -> Integer로 자동 변환 (오토박싱)`
```
- 내부적으로는 `Integer.valueOf(x)` 호출.
---

### ✅ 언박싱 (Unboxing)
- **래퍼 클래스 객체를 자동으로 기본형으로 변환**하는 것.
- 예:
```java
Integer a = Integer.valueOf(5); 
int b = a;  // Integer -> int로 자동 변환 (언박싱)
```
- 내부적으로는 `a.intValue()` 호출.
---
## ⚠️ 특징과 주의사항
- 박싱/언박싱은 코드에 안 보여도 **객체 생성, 메서드 호출 등의 비용**이 숨어 있다.
- 반복 연산이나 대량 데이터 처리에서는 **메모리 사용과 성능 저하**가 누적될 수 있다.
- `Integer` 범위 -128 ~ 127은 **캐싱**되어 재사용되지만, 그 외 값은 새 객체가 생성된다.
- 그래서 Java에서는 성능 최적화를 위해 **기본형 특화 함수형 인터페이스** (`IntFunction`, `IntPredicate` 등)를 제공한다.
