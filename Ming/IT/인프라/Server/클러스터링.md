> 여러 대의 서버(WAS 등)를 묶어서 하나의 시스템처럼 동작하게 만드는 기술

---
## 🧩 클러스터링에서 주로 다루는 것
### 1. 세션 클러스터링
> 사용자의 세션 정보를 여러 서버가 공유하거나 복제
- 예) Tomcat A에서 로그인한 사용자 → Tomcat B에서도 로그인 상태 유지
- 구현 방법: **세션 복제**, **세션 공유(예: Redis)**
### 2. 캐시 클러스터링
> 서버 간 캐시 데이터 공유 (예: 분산 캐시 시스템)
- 구현 예: Ehcache, Hazelcast, Redis, Memcached
### 3. 메시지 클러스터링
> Kafka, RabbitMQ 같은 메시지 시스템의 노드 다중화

---
## 🔁 클러스터링 방식 2가지

|방식|설명|예시|
|---|---|---|
|**복제 기반 (Replication)**|노드 간 직접 데이터를 복사|Tomcat `DeltaManager`, `BackupManager`|
|**공유 저장소 기반 (Shared Storage)**|외부 저장소(Redis, DB 등)에 저장|Redis 세션, DB 기반 세션|

---
## 🎯 실무에서 가장 많이 쓰는 조합

| 프레임워크              | 클러스터링 방식                      | 비고                   |
| ------------------ | ----------------------------- | -------------------- |
| Spring Boot        | Redis 세션 저장소                  | Spring Session 사용    |
| Tomcat + Apache    | DeltaManager or BackupManager | mod_jk + jvmRoute 필요 |
| Spring + Hazelcast | 세션, 캐시 복제 모두 가능               | 고급 사용                |
| Spring Cloud       | Redis, Eureka, Consul 등과 통합   | 마이크로서비스 기반           |
