```kotlin
fun main() {  
    println(Counter.cnt)  
    Counter.countUp()  
    Counter.countUp()  
  
    println(Counter.cnt)  
    Counter.clear()  
    println(Counter.cnt)  
  
  
    val a = FoodPoll("짜장")  
    val b = FoodPoll("짬뽕")  
  
    a.vote()  
    a.vote()  
    b.vote()  
    b.vote()  
    b.vote()  
  
    println("${a.name} : ${a.count}")  
    println("${b.name} : ${b.count}")  
    println("총계 : ${FoodPoll.total}")  
}  
  
object Counter {  
    var cnt = 0  
    fun countUp() {  
        cnt++  
    }  
    fun clear() {  
        cnt = 0  
    }  
}  
  
class FoodPoll(val name: String) {  
    companion object {  
        var total = 0  
    }  
    var count = 0  
    fun vote() {  
        total++  
        count++  
    }  
}
```
###  `object` (Singleton: 싱글톤)
> 클래스 선언과 동시에 **단 하나의 인스턴스**를 생성
- **특징:** 앱이 실행되는 동안 메모리에 딱 하나만 존재하며, 어디서든 이름으로 바로 접근할 수 있다
###  `companion object` (동반 객체)
> 특정 클래스 내부에 선언되어, **해당 클래스와 운명을 함께하는 객체**
- 자바의 `static`처럼 보이지만, 실제로는 클래스 안에 속한 **객체**로 클래스당 단 하나만 가질 수 있다.
