```kotlin
fun main() {  
    var price = 5000  
    // apply: 인스턴스 생성후 변수에 담기전 초기화 과정을 진행할 때 많이 사용함  
    val book = Book("책", 2).apply {  
        name = "[초특가]" + name  
        discount()  
    }  
    println(book.name)  
  
    // run: 계싼이 필요하거나 개게 가공 후 결과 리턴이 필요할 떄  
    val run = book.run {  
        "test${price}"  
    }  
    println("run : ${run}")  
  
    // with: run과 거의 같지만 호출 방식이 다름  
    val with = with(book) {  
        "TEST ${name}"  
    }  
    println("with : ${with}")  
  
    // run과 같지만 it을 통해 인스턴스 접근이 가능하다.  
    // ** main의 변수와 동일 변수명일 경우 혼란을 방지하기 위해 사용 가능하다.    
    val let = book.let {  
        "Test${it.price}"  
    }  
    println("let : ${let}")  
  
    // apply함수의 let 버전  
    val also = book.also {  
        it.name = "apply의 let버전 also : ${it.name}"  
    }  
    println(also.name)  
  
}  
  
class Book(var name: String, var price: Int) {  
    fun discount() {  
        price -= 2000  
    }  
}
```
```console
[초특가]책
run : test5000
with : TEST [초특가]책
let : Test-1998
apply의 let버전 also : [초특가]책
```

|**함수**|**객체 접근 (receiver)**|**반환값 (return)**|**주요 사용처**|
|---|---|---|---|
|**`apply`**|**`this`** (생략 가능)|**객체 본인**|객체 생성 직후 초기화할 때|
|**`also`**|**`it`** (이름 변경 가능)|**객체 본인**|객체 확인(로그 출력), 부수적인 작업|
|**`run`**|**`this`** (생략 가능)|**마지막 문장**|계산이 필요하거나 객체 가공 후 결과 리턴|
|**`let`**|**`it`** (이름 변경 가능)|**마지막 문장**|`null` 체크 (`?.let`), 지역 변수 대체|
|**`with`**|**`this`** (생략 가능)|**마지막 문장**|이미 생성된 객체의 속성을 여러 개 사용할 때|
