## 오버라이드
``` kotlin
fun main() {  
    var a = Tiger()  
    a.eat()  
}  
  
open class Animal2 {  
    open fun eat() {  
        println("음식을 먹습니다")  
    }  
}  
  
class Tiger : Animal2() {  
    override fun eat() {  
        println("고기를 먹습니다.")  
    }  
}  
```

## 추상클래스
```kotlin
fun main() {  
    // 추상 클래스  
    var b = Rabbit()  
    b.eat()  
    b.sniff()  
}  
  
abstract class Animal3 {  
    abstract fun eat()  
    fun sniff() {  
        println("킁킁")  
    }  
}  
  
class Rabbit : Animal3() {  
    override fun eat() {  
        println("당근을 먹습니다.")  
    }  
}
```

## 인터페이스
1. 코틀린에서 속성,추상함수,일반함수를 가질 수 있다
2. 인터페이스에서 생성자를 가질 수 없다
3. 인터페이스 구현부가 있는 함수는 open 함수로 간주
4. 인터페이스 구현부가 없는 함수는 abstract함수로 간주
```kotlin
fun main() {  
  
    var d = Dogs()  
    d.run()  
    d.eat()  
}  
  
interface Runner {  
    fun run()  
}  
  
interface  Eater {  
    fun eat() {  
        println("dmatlrdmf ajrtmqslek")  
    }  
}  
  
class Dogs : Runner, Eater {  
    override fun run() {  
        println("달린다")  
    }  
  
    override fun eat() {  
        println("먹는다")  
    }  
}
```
