## 1. 코루틴의 본질 및 패키지 구조
- **정체**: 스레드를 직접 차단(Blocking)하지 않고, 작업을 잠시 중단(Suspend)했다가 결과가 오면 다시 재개(Resume)하는 '경량 스레드' 모델임.
- **패키지 분리**:
    - **`kotlin.coroutines`**: 언어 수준의 최소한의 뼈대이며 `suspend` 키워드 해석 등 핵심 규격을 포함함.
    - **`kotlinx.coroutines`**: 실무용 API(`launch`, `async` 등)를 담은 확장 라이브러리이며 별도 의존성 추가가 필요함.
## 2. 3대 핵심 구성 요소
- **`CoroutineScope`**: 코루틴이 실행되는 유효 범위이며, 부모 스코프가 종료되면 하위 코루틴도 일괄 취소됨 (구조적 동시성).
- **`CoroutineContext`**: 코루틴의 이름, 작업 제어(Job), 실행 스레드(Dispatcher) 등 설정값의 집합임.
- **`Dispatcher`**: 작업을 실행할 스레드 풀을 결정함.
    - **`Dispatchers.IO`**: DB, 네트워크 등 I/O 작업용 (스레드 수가 많음).
    - **`Dispatchers.Default`**: CPU 연산 집약적인 작업용 (코어 수에 비례).
## 3. 코루틴 빌더 (Builders)
- **`runBlocking`**: 현재 스레드를 차단하며 코루틴이 끝날 때까지 대기함. 서버 로직 내 사용은 금지되며, 오직 `main`진입점이나 테스트 코드에서만 사용함.
- **`launch`**: 결과값이 필요 없는 비동기 작업을 수행하며 `Job` 객체를 반환함. 자바의 `Runnable` 실행과 유사함.
- **`async`**: 결과값이 필요한 비동기 작업을 수행하며 `Deferred<T>`를 반환함. `await()`를 통해 결과값을 받아낼 수 있음.
## 4. 제어 함수 및 스레드 스위칭
- **`join()`**: 작업이 완료될 때까지 현재 코루틴을 중단하고 대기함 (결과값 없음).
- **`await()`**: 작업이 완료될 때까지 대기한 후 결과값을 반환함.
- **`withContext()`**: 현재 코루틴을 유지한 채로 내부 블록만 특정 `Dispatcher`로 스레드 풀을 교체하여 실행함.
## 5. 결정적 주의사항: `run` vs `launch`
- **`pool.run { }`**: 코루틴 빌더가 아닌 **스코프 함수**임. 새로운 코루틴을 만들지 않고 현재 스레드(Main 등)에서 동기적으로 실행되므로 스레드 풀 효과가 없음.
- **`launch(pool) { }`**: 실제 코루틴 빌더이며 지정된 스레드 풀에서 비동기로 작업을 수행함.
- **비차단 특성**: 코루틴의 `delay()`는 스레드를 점유하지 않고 풀에 즉시 반납함. 따라서 적은 수의 스레드로도 수많은 작업을 병렬처럼 처리할 수 있음.
## 6. 실무 적용 가이드
- **스레드 제한**: 특정 작업의 스레드 개수를 제한하려면 자바의 `Executors`를 생성한 뒤 **`.asCoroutineDispatcher()`**로 변환하여 사용함.
- **서버 환경**: Spring Boot(WebFlux) 등에서는 프레임워크가 스코프를 제공하므로 `runBlocking` 없이 **`suspend fun`**을 전파하여 로직을 짬.
- **의존성**: 인텔리제이에서 `kotlinx` 패키지가 인식되지 않으면 `build.gradle`에 반드시 코루틴 라이브러리를 추가하고 리로드해야 함.
```kotlin
import kotlinx.coroutines.*  
import java.util.concurrent.Executors  
  
fun main() {  
    // GlobalScope  
    runBlocking {  
        launch {  
            poolCoroutin()  
        }  
        defaultCoroutin()  
    }  
  
}  
  
suspend fun poolCoroutin() = coroutineScope {  
    Executors.newFixedThreadPool(4).asCoroutineDispatcher().use { pool ->  
        repeat(10) { i ->  
            launch(pool) {  
                println("[${Thread.currentThread().name}]POOL 작업 진행 합니다. [${i}]")  
                delay(2000)  
                println("[${Thread.currentThread().name}]POOL 작업 완료 [${i}]")  
            }  
        }    }  
}  
  
suspend fun defaultCoroutin() = coroutineScope {  
    println("ASYNC START")  
    val async = async(Dispatchers.Default){  
        for (i in 1..10) {  
            delay(1000)  
            println("딜레이")  
  
        }  
        "FISNISH"  
    }  
  
    val launch = launch(Dispatchers.Default){  
        for (i in 1..10) {  
            delay(500)  
            println("런치 딜레이")  
  
        }  
        "FISNISH"  
    }  
    println(async.await())  
    launch.join()  
}
```
