### 상황
- 서버는 소켓을 하나만 생성되게끔 로직이 만들어져 있다. 
	- 즉, `ServerSocket.accept()`가 하나의 스레드에서만 실행된다.
- 클라이언트는 A와 B 두개의 클라이언트가 서버와 연결을 시도한다.
	- A클라이언트는 서버와 연결후 데이터를 주고 받는다.
	- B클라이언트는 서버와 연결은 되지만 데이터를 주고 받을 수 없다.

- OS 계층 : TCP 3 way handshake가 발생하고 클라이언트들(A,B)은 서버와 TCP 연결이 된다.
> 서버의 소켓(서버 소켓 아님)은 생성되지 않음.


### 2. 소켓 생성
> A클라이언트는 소켓을 생성한다.  B클라이언트는 서버와 연결만 되어있다.
![[Pasted image 20250607202750.png]]
- 서버가 클라이언트와 데이터를 주고 받으려면 소켓을 획득해야 한다.
- `ServerSocket.accept()` 메서드를 호출하면 BackLog 큐의 정보를 기반으로 소켓 객체를 하나 생성한다.
- 큐 이므로 순서대로 데이터를 꺼낸다.

### 3. 결과
![[Pasted image 20250607203545.png]]
- A클라이언트는 데이터를 주고 받을 수 있다. 소켓이 연결되었기 때문.
- B클라이언트는 데이터를 서버로 보낼 수 있어도 소켓이 생성되지 않았기 때문에 서버로부터 응답 받을 수 없다.
