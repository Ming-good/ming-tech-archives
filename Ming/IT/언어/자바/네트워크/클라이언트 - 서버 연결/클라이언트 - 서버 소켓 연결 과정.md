### Client
```java
Socket sock = new Socket("localhost", PORT);  
DataInputStream input = new DataInputStream(sock.getInputStream());  
DataOutputStream output = new DataOutputStream(sock.getOutputStream());
```
- Socket : 서버와 데이터를 주고 받기 위한 스트림을 제공
- InputStream : 서버에서 클라이언트로 전달한 데이터를 받을 때 사용
- OutputStream : 클라이언트에서 서버에 데이터를 전달할 때 사용

### Server
![[Pasted image 20250603225017.png]]
-  연결 시도시 OS 계층에서 TCP 3 way handshake가 발생하고 TCP연결 된다.
- TCP 연결이 완료되면 서버는 **OS backlog queue**라는 곳에 클라이언트와 서버의 TCP 연결정보를 보관한다.

##### 클라이언트와 랜덤 포트
TCP연결시에는 클라이언트 서버 모두 IP와 포트 정보가 필요하다
- **클라이언트** : localhost, 50000(포트 랜덤 생성)
- **서버** : localhost, 12345

```java
ServerSocket server = new ServerSocket(PORT);  
Socket socket = server.accept(); // 클라이언트가 접속 될 때 까지 기다림.  
  
DataInputStream input = new DataInputStream(socket.getInputStream());  
DataOutputStream output = new DataOutputStream(socket.getOutputStream());
```
- **서버 소켓은 단지 클라이언트와 서버의 TCP연결만 지원하는 특별한 소켓이다.**
- 클라이언트와 서버가 정보를 주고 받으려면 `Socket` 객체가 필요하다.(서버 소켓 아님)

##### accept() 호출 과정
![[Pasted image 20250603230403.png]]

1. `accept()` 호출시 backlog queue에서 TCP연결 정보를 조회
	- 만약 TCP 연결 정보가 없다면, 연결 정보가 생성될 때 까지 대기(블로킹)
2. 해당 정보를 기반으로 `Socket` 객체 생성
3. 사용한 TCP연결정보는 backlog queue에서 제거된다
