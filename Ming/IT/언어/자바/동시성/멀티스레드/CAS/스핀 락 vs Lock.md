### 구현
```java
public void lock() {
	log("락 획득 시도");
		// 락을 획득할 때 까지 스핀 대기(바쁜 대기) 한다.
		while (!lock.compareAndSet(false, true)) {
		log("락 획득 실패 - 스핀 대기");
	}
	log("락 획득 완료");
}
```


### 스핀 락 장점
1. 원자적인 연산은 스레드 입장에서 쪼갤 수 없는 하나의 연산이다. 따라서 여러 스레드가 동시에 실행해도 안전하다.
2. CAS를 사용해서 원자적인 연산을 만든 덕분에 무거운 동기화 작업 없이 아주 가벼운 락을 만들 수 있다. 
3. Lock방식은 컨텍스트 스위칭으로 인하여 성능이 상대적으로 느릴 수 있다. 반면에 CAS를 활용한 락 방식은 사실 락이 없다. 
4. 단순히 while문을 반복할 뿐이다. 따라서 대기하는 스레드도 `RUNNABLE`상태를 유지하면서 가볍고 빠르게 작동할 수 있다.

### 스핀 락 단점
```java
// Main 실행 클래스
public void run() {
	spinLock.lock();
	try {
		// 임계 영역
		log("비즈니스 로직 실행");
		// 오래 걸리는 로직에서 스핀 락 사용X (1ms도 오래걸리는 작업이다.)
		sleep(1); 
	} finally {
		spinLock.unlock();
	}
}
```
> 락을 기다리는 스레드가 `BLOCKED` , `WAITING` 상태로 빠지지는 않지만, `RUNNABLE` 상태로 락을 획득할 때 까지 while문을 반복하는 문제가 있다. (CPU 자원을 계속 사용하는 것이다.)

- 동기화 락을 사용하면 `RUNNABLE` 상태의 스레드가 `BLOCKED` , `WAITING` 상태에서 다시 `RUNNABLE` 상태로 이동한다. 이 사이에 CPU 자원을 거의 사용하지 않을 수 있다.

### 스핀 락의 사용처
- 안전한 임계 영역이 필요하지만, 연산이 길지 않고 **매우매우매우! 짧게 끝날 때** 사용해야 한다.
- 숫자 값의 증가, 자료 구조의 데이터 추가와 같이 CPU 사이클이 금방 끝나는 연산에 사용하면 효과적이다.
> 반면에 데이터베이스의 결과를 대기한다거나, 다른 서버의 요청을 기다린다거나 하는 것 처럼 오래 기다리는 작업에 사용하면 CPU를 계속 사용하며 기다리는 최악의 결과가 나올 수도 있다.
