
# Enum의 역사
#### 1. 열거 타입의 등장
- Enum이 등장하기 이전 상수값 관리를 [[정수형 열거 패턴]]으로 했었습니다. 
-  당시 컴퓨터의 성능이 좋지 않았기 때문에 `equals` 보다는 \==로 비교를 했었음. 
-  정수 열거 패턴의 단점은 안전을 보장할 수 없으며, 표현력도 좋지 않았습니다.

#### 2. Enum의 기능
- **final String name()** : 상수의 이름을 반환합니다.
- **final int ordinal() :** 상수의 열거형에서의 순서를 반환합니다.
- **String toString()** : 열거형 상수의 문자열 표현을 반환합니다. 재정의 가능합니다.
- **final boolean equals(Object obj)** : 매개변수로 전달된 객체가 열거형 상수와 같다면 true를 반환하고, 그렇지 않다면 false를 반환합니다.
- **final int hashCode()** : 이 열거형 상수에 대한 해시 코드를 반환합니다. 실제 구현은 super.hashCode()를 호출합니다.
- **final int compareTo(E obj)** : 열거형의 순서를 비교합니다. 순서가 낮은 경우 음수, 같은 경우 0, 높은 경우 양수를 반환합니다.
- **final Class `<E> getDeclaringClass()`** : 열거형 상수의 타입에 해당하는 Class 객체를 반환합니다.
- **final Object clone()** : 열거형이 복제되지 않도록 보장하며, 단일 상태임을 보장해줍니다. 사용 시CloneNotSupportedException이 발생합니다. 열거형 상수를 만들기 위해 컴파일러 내부적으로 사용된다고 합니다.
- **final void finalize() :** enum 클래스가 finalize 메서드를 가질 수 없음을 보장해줍니다.

# Enum 요약
#### 🔸 1. `enum`이란?
- 자바 1.5부터 도입된 **열거형 타입**
- **고정된 상수 집합을 표현하는 특별한 클래스**
- 내부적으로는 `Enum<T>` 클래스를 **상속한 클래스**로 컴파일됨
#### 🔸 2. `enum`은 객체다!
- 각 상수는 사실 `static final` 인스턴스 객체
- == 비교 가능 (주소값으로 비교 하기 때문)
- `ordinal()` 메서드를 통해 **0부터 시작하는 정수값** 제공 (순서)
#### 🔸 3. `switch`에서 사용 가능 (자바 1.5부터)
- `switch(enum)`은 내부적으로 **`ordinal()`을 기반으로 컴파일**
- 참조값으로 비교하지 않고, **정수값 매핑 테이블을 통해 분기**
#### 🔸 4. `enum`은 동적으로 생성되지 않는다 (cglib ❌)
- 컴파일 타임에 `.class`로 고정 생성됨
- `Enum.valueOf()`, `values()`는 컴파일러가 자동 생성

| 비교 항목     | enum         | int             | String          |
| --------- | ------------ | --------------- | --------------- |
| 타입 안정성    | ✅ 높음         | ❌ 없음            | ❌ 없음            |
| 비교 성능     | ✅ ==로 빠름     | ✅ 빠름            | ❌ `equals()` 호출 |
| 가독성       | ✅ 좋음         | ❌ 나쁨 (숫자 의미 모름) | ✅ 이름 보임         |
| 확장성       | ✅ 메서드, 필드 가능 | ❌ 불가            | ❌ 불가            |
| switch 지원 | ✅ (1.5부터)    | ✅               | ✅ (1.7부터)       |
