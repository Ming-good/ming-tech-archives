##### 정상 종료 - 셧다운 훅 실행 됨
- 모든 non 데몬 스레드의 실행 완료로 자바 프로세스 정상 종료
- 사용자가 Ctrl+C를 눌러서 프로그램을 중단
- `kill` 명령 전달 (`kill -9` 제외)
- IntelliJ의 stop 버튼

##### 강제 종료 - 셧다운 훅 실행 안 됨
- 운영체제에서 프로세스를 더 이상 유지할 수 없다고 판단할 때 사용
- 리눅스/유닉스의 kill -9나 Windows의 taskkill /F

> **정상 종료의 경우 셧다운 훅이 작동**해서 프로세스 종료 전에 필요한 후 처리가 가능하지만 
> **강제 종료의 경우 셧다운 훅이 작동하지 않는다.**


### 구현 방법
```java
Runtime.getRuntime().addShutdownHook(new Thread(shutdownHook, "shutdown"));
```

### 자원 정리 대기 이유
```java
Thread.sleep(1000)
```
보통 모든 non-데몬 스레드의 실행이 완료되면 자바 프로세스가 정상 종료된다.
하지만 아래의 경우 non-데몬 스레드의 종료 여부와 관계없이 자바 프로세스는 종료된다.
- 사용자가 ctrl+C를 눌러서 프로그램 중단
- kill 명령 전달 (kill -9 제외)
- IDE stop버튼
> 셧다운 훅의 실행 끝날 때 까지는 기다려주지만 셧다운 훅의 실행이 끝나면 
> non-데몬 스레드의 실행 여부과 상관 없이 자바 프로세스는 종료된다.
> 따라서 다른 스레드가 자원을 정리하거나 필요한 로그를 남길 수 있다록 셧다운 훅의 실행을 잠시 대기한다.
