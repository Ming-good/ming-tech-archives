- 리액티브 시스템을 구축하는 데 필요한 프로그래밍 모델
## 특징
1. [[선언형 프로그래밍]]으로 동작을 구체적으로 명시하지 않고 목표만 선언한다.
2. 데이터가 지속적으로 발생하며 데이터가 발생하는 것을 이벤트로 보고, 이 이벤트를 발생시키면서 데이터를 계속적으로 전달하는 것을 의미함.
3. 데이터 소스 변경이 있을 때마다 데이터를 전파한다.

## 구성
1. `Publisher`(생산자):  입력으로 들어오는 데이터를 제공하는 역할
2. `Subscriber`(소비자): `Publisher`가 제공한 데이터를 전달받아서 사용하는 주체
3. `Data Source`: `Publisher`의 입력으로 전달되는 데이터
4. `Operator`: 데이터 가공 처리 담당
