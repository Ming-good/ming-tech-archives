> 데이터 스트림을 논블로킹이면서 비동기적인 방식으로 처리하기 위한 리액티브 라이브러리 표준 사양

- 리액티브 스트림즈 구현체: RxJava, Reactor, Akka Streams, Java9 Flow API

## 리액티브 스트림즈 구성요소
| 컴포넌트         | 설명                                                                                                                 |
| ------------ | ------------------------------------------------------------------------------------------------------------------ |
| Publisher    | 데이터를 생성하고 **통지(발행, 게시, 방출)하는 역할**을 한다.                                                                             |
| Subscriber   | 구독한 Publisher로부터 통지(발행, 게시, 방출)된 데이터를 전달받아서 **처리하는 역할**을 한다.                                                       |
| Subscription | Publisher에 요청할 **데이터의 개수를 지정**하고, 데이터의 **구독을 취소**하는 역할을 한다.                                                        |
| Processor    | Publisher와 Subscriber의 기능을 모두 가지고 있다. 즉, Subscriber로서 다른 Publisher를 구독할 수 있고, Publisher로서 다른 Subscriber가 구독할 수 있다. |

## `Publisher`
```java
public interface Publisher<T> {  
	public void subscribe(Subscriber<? super T> s);  
}
```
- `subscribe(subscriber)`: 파라미터로 전달받은 `Subscriber`를 등록하는 역할
> Kafka의 Pub/Sub모델은 느슨한 결합 구조로 토픽을 통해 구독과 데이터 전송을 받지만, 리액티브 스트림즈에서는 개념상으로는 `Subscriber`가 구독하는 것이 맞는데 `Publisher`가 파라미터인 Subscriber를 등록하는 형태로 구독이 이루어짐.

## `Subscriber`
```java
public interface Subscriber<T> {  
	public void onSubscribe(Subscription s);  
	public void onNext(T t);  
	public void onError(Throwable t);  
	public void onComplete();  
}
```
- `onSubscribe(Subscription s)`: 요청 데이터 개수 지정, 구독 해지 같은 처리하는 역할
- `onNext(T t)`: `Publisher`가 통지한 데이터를 처리하는 역할
- `onError(Throwable t)`: 에러 발생시 해당 에러를 처리하는 역할
- `onComplete()`: `Publisher`가 데이터 통지를 완료했음을 알릴 때 호출되는 메서드로 완료후 후처리가 필요하다면 `onComplete`에서 처리 코드를 작성하면 된다.

## `Subscription`
```java
public interface Subscription {  
  
	public void request(long n);  
	public void cancel();  
}
```
- `Subscriber`가 `Publisher`로 요청할 때 사용되는 객체
- `Publisher`에게 데이터 개수 요청하거나 `cancel`메서드를 통해 구독을 해지할 수 있다.

## `Processor`
```java
public interface Processor<T, R> extends Subscriber<T>, Publisher<R> {  
}
```
- `Processor`는 `Publisher`와 `Subscriber` 기능을 모두 가지고 있다. 따라서 두 인터페이스를 상속한다.
