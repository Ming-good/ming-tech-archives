1. **`concatWith(Publisher<? extends T> other)`**
	- `Publisher`(Flux/Mono)를 **순차적으로 연결**(concat)해서 실행하는 연산자
	-  에러 발생시 뒤의 Publisher는 실행되지 않음
	-  순서 보장(순차적으로 실행된다 앞의 Publisher실행후 뒤의 Publisher를 구독함)
	-  인스턴스 메서드
2. **`delayElements(Duration delay)`**
	- 데이터 소스로 입력된 각 데이터의 emit을 일정시간동안 지연시키는 연산자
3. **`share()`**(Warm up)
	- Cold Sequence를 Hot Sequence로 동작하게 해주는 연산자
	- 여러 Subscriber가 하나의 원본 Flux를 공유하는 연산자
4. **`cache()`**(Hot)
	- Cold Sequence로 동작하는 Mono를 Hot Sequence로 변경 해주는 연산자 
	- emit된 데이터를 캐시한 뒤, 구독이 발생할 때마다 캐시된 데이터를 전달함.
5. `doOnNext()`
	- 데이터가 emit될 때마다 부수작업을 수행하는 연산자
	- 원본 데이터 흐름에는 영향을 주지 않음(디버깅,로깅,모니터링 용도)
6. `doOnSubscribe()`
	- 구독이 발생한 시점에 추가적인 처리가 필요할 경우 동작 추가
---
```java
System.out.println("=== FLUX EX3 ===");  
Flux<String> flux = Mono.justOrEmpty("Steve")  
	.concatWith(Mono.justOrEmpty("Jobs"));  
flux.subscribe(System.out::println);

System.out.println("=== FLUX EX4 ===");  
Flux.concat(  
	Flux.just("mercury", "Venus", "Earth"),  
	Flux.just("Mars", "Jupiter", "Saturn"),  
	Flux.just("Uranus", "Neptune", "Pluto")  
).collect(Collectors.toList())  
.subscribe(System.out::println);
```

```console
=== FLUX EX3 concatWith() ===
Steve
Jobs

=== FLUX EX4 ===
[mercury, Venus, Earth, Mars, Jupiter, Saturn, Uranus, Neptune, Pluto]
```

#### 필터링 연산자
1. `filter()`: UpStream에서 emit된 데이터 중에서 조건에 일치하는 데이터만 DownStream으로 emit
2. `filterWhen(() -> Mono.just())`: 비동기적으로 필터링을 수행
3. `skip(n)`
	- Upstream에서 emit된 데이터 중에서 파라미터로 입력받은 숫자만큼 스킵
	- 파라미터로 시간을 지정하면 지정된 시간만큼 emit된 데이터를 스킵 
		- EX)`skip(Duration.ofSeconds(1))`
4. `take(n)`
	- 입력받은 숫자만큼 Upstream에서 emit되는 데이터를 Downstream으로 emit한다.
	- 파라미터로 시간을 지정하면 지정된 시간만큼 emit된 데이터를 Downstream으로 emit한다.
		- EX) `take(Duration.ofSeconds(1))`
5. `takeLast(n)`
	- Upstream에서 emit된 데이터 중에서 파라미터로 입력한 개수만큼 마지막에 emit된 데이터를 Downstream으로 emit한다.
6. `takeUntil()`
	- 파라미터로 입력한 표현식이 `false`면 Upstream에서 emit된 데이터를 Downstream으로 emit한다.
	- `true`이면 Sequence가 종료된다.
7. `takeWhile()`
	- 파라미터로 입력한 표현식이 `true`면 Upstream에서 emit된 데이터를 Downstream으로 emit한다.
	- `false`이면 Sequence가 종료된다.
8. `next()`
	- Upstream에서 emit되는 데이터 중에서 첫 번째 데이터만 Downstream으로 emit한다.

#### 조합 연산자
1. `zip(Flux, FLux)`: **여러 Flux/Mono를 한꺼번에 결합**하는 **정적 메서드** (결과는 `Flux<Tuple2>` 또는 매퍼 함수 결과)
2. `zipWith(Flux, (Flux, Flux))`: **자기 자신과 다른 Flux/Mono를 체이닝 방식으로 결합**하는 **인스턴스 메서드** (보통 2개 결합할 때 간단하게 사용)
3. **`concat(Flux.just(), Flux.just())`**
	- 순서대로 처리된다.
	- 여러개의 Publisher를 순차적으로 연결해서 실행하는 연산자
4. `flatMap()`
	- 전달되는 데이터 수는  (Upstream에서 emit되는 데이터 수) * (Inner Sequence에서 emit되는 데이터수) 이다
	- 순서 상관없이 먼저 처리되는 데이터가 비동기적으로 Downstream으로 emit된다.
5. `merge(Flux.just(), Flux.just())`
	- emit된 데이터를 인터리빙 방식으로 병합함
	- emit되는 데이터는 번갈아가면서 merge하는 것이 아닌 emit된 시간 순서대로 merge를 진행한다.
