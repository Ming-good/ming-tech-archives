1. **`concatWith(Publisher<? extends T> other)`**
	- `Publisher`(Flux/Mono)를 **순차적으로 연결**(concat)해서 실행하는 연산자
	-  에러 발생시 뒤의 Publisher는 실행되지 않음
	-  순서 보장(순차적으로 실행된다 앞의 Publisher실행후 뒤의 Publisher를 구독함)
	-  인스턴스 메서드
2. **`concat()`**
	- 정적 메서드
	- 여러개의 Publisher를 순차적으로 연결해서 실행하는 연산자
3. **`delayElements(Duration delay)`**
	- 데이터 소스로 입력된 각 데이터의 emit을 일정시간동안 지연시키는 연산자
4. **`share()`**(Warm up)
	- Cold Sequence를 Hot Sequence로 동작하게 해주는 연산자
	- 여러 Subscriber가 하나의 원본 Flux를 공유하는 연산자
5. **`cache()`**(Hot)
	- Cold Sequence로 동작하는 Mono를 Hot Sequence로 변경 해주는 연산자 
	- emit된 데이터를 캐시한 뒤, 구독이 발생할 때마다 캐시된 데이터를 전달함.
6. `doOnNext()`
	- 데이터가 emit될 때마다 부수작업을 수행하는 연산자
	- 원본 데이터 흐름에는 영향을 주지 않음(디버깅,로깅,모니터링 용도)
7. `doOnSubscribe()`
	- 구독이 발생한 시점에 추가적인 처리가 필요할 경우 동작 추가
---
```java
System.out.println("=== FLUX EX3 ===");  
Flux<String> flux = Mono.justOrEmpty("Steve")  
	.concatWith(Mono.justOrEmpty("Jobs"));  
flux.subscribe(System.out::println);

System.out.println("=== FLUX EX4 ===");  
Flux.concat(  
	Flux.just("mercury", "Venus", "Earth"),  
	Flux.just("Mars", "Jupiter", "Saturn"),  
	Flux.just("Uranus", "Neptune", "Pluto")  
).collect(Collectors.toList())  
.subscribe(System.out::println);
```

```console
=== FLUX EX3 concatWith() ===
Steve
Jobs

=== FLUX EX4 ===
[mercury, Venus, Earth, Mars, Jupiter, Saturn, Uranus, Neptune, Pluto]
```

#### 필터링 연산자
1. `filter()`: UpStream에서 emit된 데이터 중에서 조건에 일치하는 데이터만 DownStream으로 emit
2. `filterWhen()`: 비동기적으로 필터링을 수행
