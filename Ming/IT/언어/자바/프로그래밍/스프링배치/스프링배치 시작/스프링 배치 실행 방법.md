## 1. **CommandLineJobRunner**
- **역할**
    - Spring Batch **전통적인 실행 방식**.
    - `main()` 메서드를 가진 클래스로, CLI(Command Line)에서 직접 Job을 실행할 때 사용.
    - XML 기반 설정을 많이 쓰던 시절에 주로 사용됨.
- **사용법**
    `java -cp my-batch.jar org.springframework.batch.core.launch.support.CommandLineJobRunner com.example.batch.MyJobConfig myJobName date=20250909`
    - 첫 번째 인자: 스프링 배치 Job 설정이 담긴 XML 파일
    - 두 번째 인자: 실행할 Job 이름
    - 이후 인자: JobParameters
- **특징**
    - `ApplicationContext`를 직접 띄워서 Job을 실행.
    - Spring Boot 등장 이전 방식.
    - 지금은 Boot 환경에서는 잘 안 씀(레거시 프로젝트에서 볼 수 있음).

## 2. **JobLauncherApplicationRunner**
- **역할**
    - **Spring Boot 환경에서 Job 실행을 담당하는 Runner**.
    - Boot의 `ApplicationRunner` 구현체 → Spring Boot 애플리케이션이 시작될 때 자동으로 동작.
    - `spring.batch.job.name` 프로퍼티에 지정된 Job을 찾아 실행.
- **사용법** (Spring Boot `application.yml` / `application.properties`)
    ```yaml
    spring:   
	    batch:     
		    job:       
			    name: myJob
    ```
    - 애플리케이션 실행 시 `myJob` 자동 실행.
- **특징**
    - Boot의 라이프사이클과 자연스럽게 통합됨.
    - `@EnableBatchProcessing` 또는 Spring Boot Batch AutoConfiguration과 함께 사용.
    - 여러 Job이 정의돼 있더라도, `spring.batch.job.name`으로 지정된 Job만 실행.
    - JobParameters도 `--param=value` 형태로 CLI에서 전달 가능.
	    `java -jar batch-app.jar --spring.batch.job.name=myJob requestDate=2025-09-07`
