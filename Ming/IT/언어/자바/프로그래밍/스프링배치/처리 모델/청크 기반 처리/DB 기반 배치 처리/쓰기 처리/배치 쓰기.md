## `JdbcBatchItemWriter` 구조

![[Pasted image 20251016151553.png]]
- **`NamedParameterJdbcTemplate`**
	- 네임드 파라미터 바인딩을 지원하는 `JdbcTemplate`로 SQL 실행과 결과를 처리하기 위해 사용된다.
- **`SQL`**
	- `JdbcBatchItemWriter`가 실행할 SQL 구문을 정의한다
	- 지원하는 파라미터 바인딩
		- 물음표(?) 플레이스홀더
		- 네임드 파라미터(`:name`)
- **`ItemSqlParameterSourceProvider` - 네임드 파라미터 매핑의 해결사**
	- 내부 `BeanPropertyItemSqlParameterSourceProvider` 구현체를 통해 자바 리플렉션 기반으로 필드값을 바인딩한다.

## 사용 예시
```java
@Bean
public JdbcBatchItemWriter<HackedOrder> orderStatusWriter() {
	return new JdbcBatchItemWriterBuilder<HackedOrder>()
			.dataSource(dataSource)
			.sql("UPDATE orders SET status = :status WHERE id = :id")
			.beanMapped()
			.assertUpdates(true)
			.build();
}
```
- `beanMapped()`
	- Order 객체의 필드를 SQL 파라미터에 자동으로 매핑한다.
- `assertUpdates()`
	- `true`(기본값)
		- 단 하나의 데이터라도 업데이트에 실패하면 예외를 던져 중단 한다
		- 모든 데이터가 반드시 정상화되어야 하는 경우 사용
	- `false`
		- 일부 데이터가 업데이트 실패되어도 계속 진행된다.
		- 중복 데이터 처리구문을 사용할 때나 조건부 `UPDATE`처럼 일부 데이터는 변경되지 않아도 되는 상황에서 사용
