## `JobInstance`
- **`Job`의 논리적 실행 단위**로 `Job`이 "어떤 작업을 할지"를 정의하는 설계도라면, `JobInstance`는 "언제, 어떤 데이터로 실행되는지"를 구체화한 실체다.
- `Job`의 name으로 어떤 `Job`에 속하는지 구분
## `JobParameters`
- 동일한 `Job`을 **서로 다른 `JobInstance`로 구분**한다.
- `JobInstance` = `Job' + `JobParameters`이다.
- **파라미터 식별 여부 결정** : `identifying` 속성
	- 기본적으로 모든 `JobParameter`는 `identifying=true`로 설정된다.
```gradle
./gradlew bootRun --args='--spring.batch.job.name=brutalizedSystemJob \n
							chaos=true,java.lang.Boolean \n
							verbose=true,java.lang.String,false'
```
## `JobExecution`
- `JobInstance`의 실제 실행 시도이다.
- `JobExecution` 실행 정보
	- **현재 상태**: 작업이 현재 어떤 상태인지 (`COMPLETED`, `FAILED`, `STOPPED` 등)
	- **시작 시간**: Job이 실행을 시작한 시점
	- **종료 시간**: Job이 실행을 종료한 시점
	- **종료 코드**: 해당 Job 실행의 최종 결과를 나타내는 코드
	- **실패 원인**: 해당 Job 실행이 실패한 경우 해당 이유에 대한 정보
## `StepExecution`
- 단일 Step의 실행시도를 나타내는 객체이다.
- 실패, 재시작 시 
	- `StepExecution`은 `JobExecution`에 종속된다.
	- 새로운 `JobExecution`이 생성되면 그에 속하는 `StepExecution`도 새로 생성된다.
- `StepExecution` 실행정보
	- **현재 상태**: Step이 현재 어떤 상태인지를 나타내는 BatchStatus
	- **읽기/쓰기 카운트**: 성공적으로 읽거나 쓴 아이템의 수
	- **커밋/롤백 카운트**: 트랜잭션 처리 횟수
	- **스킵 카운트**: 청크 처리 중 건너뛴 아이템의 수
	- **시작/종료 시간**: Step 실행의 시간적 정보
	- **종료 코드**: Step 실행의 최종 결과 코드
	- **예외 정보**: 실패 시 발생한 오류에 대한 상세 정보
## `BatchStatus`
- `JobExecution`은 `COMPLETED`, `FAILED`, `STOPPED` 등의 실행 상태를 가지며 이상태를 `BatchStatus`라고 한다. 
- `JobExecution`과 `StepExecution`의 `BatchStatus`
	- `JobExecution`의 최종 `BatchStatus`는 해당 `JobExecution`에서 **가장 마지막에 실행된 StepExecution의 BatchStatus 값을 기준으로 결정된다**.
- `BatchStatus` 주요 상태값
	- `COMPLETED`: 배치 작업이 성공적으로 완료됨
	- `STARTING`: 배치 작업이 실행되기 직전 상태
	- `STARTED`: 배치 작업이 현재 실행 중인 상태
	- `STOPPING`: 배치 작업이 중지 요청을 받아 중지 진행 중인 상태
	- `STOPPED`: 배치 작업이 요청에 의해 중지된 상태
	- `FAILED`: 배치 작업이 실행 중 오류로 인해 실패한 상태
	- `ABANDONED`: 배치 작업이 비정상 종료되어 재시작할 수 없는 상태
	- `UNKNOWN`: 배치 작업의 상태를 확인할 수 없는 불확실한 상태
## `JobParametersIncrementer`
> 동일 `Job`을 여러 번 실행할 수 있게 해주는 특수 컴포넌트
```java
public interface JobParametersIncrementer { 
	JobParameters getNext(@Nullable JobParameters parameters); 
}
```
- 입력으로 `JobParameters`를 받아서, 변형된 `JobParameters`를 반환한다.
#### `RunIdIncrementer`
- 배치 실행마다 `run.id`라는 파라미터 값을 자동으로 증가시킨다.
```java
@Bean
public Job brutalizedSystemJob() {
    return new JobBuilder("brutalizedSystemJob", jobRepository)
            .incrementer(new RunIdIncrementer()) // 이 한 줄이 전부다💀
            .start(brutalizedSystemStep())
            .build();
}
```
