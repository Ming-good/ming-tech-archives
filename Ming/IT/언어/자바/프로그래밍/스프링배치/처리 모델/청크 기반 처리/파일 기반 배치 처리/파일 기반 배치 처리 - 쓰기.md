## `FlatFileItemWriter`
도메인 객체를 문자열로 변환해서 파일에 쓰는 역할을 한다

#### 핵심 동작
1. **필드 추출**
	-  `FieldExtractor`컴포넌트 사용
2. **문자열 결합**
	-  포맷 형식에 맞춰 결합. `LineAggregator`컴포넌트 사용
```java
// 필드 추출
Object[] fields = {victim.getName(), victim.getCauseOfDeath()};

// CSV 형식으로 문자열 결합, 예: "김배치,CPU 과부하" 
String csvRecord = Arrays.stream(fields).collect(Collectors.joining(","));
```

#### 파일 처리 옵션
- **shouldDeleteIfExists()**
	- `true`:  기존 파일이 있을 경우 삭제한다
	- `false`: 파일이 있을 경우 `ItemStreamException` 오류를 발생시킨다.
- **append()**
	- `true`: 기존 파일에 데이터 덧붙이며 `shouldDeleteIfExists`는 자동으로 `false` 상태가 된다.
	- `false`: 디폴트 상태로 덧붙이지 않는다.
- **shouldDeleteIfEmpty()**
	- 파일에 헤더와 푸터를 제외한 데이터가 하나도 쓰여지지 않았을 경우 해당 파일을 삭제할지를 결정하는 옵션이다.
	- 빈 파일 여부는 현재 스텝에서 FlatFileItemWriter가 실제로 쓴 라인 수를 기준으로 결정된다.
	- `append(true)`와 함께쓰면 기존 파일이 삭제될 수 있음으로 주의할 것
	- `true`: 빈 파일은 삭제한다
	- `false`: 디폴트로 빈 파일을 삭제하지 않는다.

#### 롤백 전략
>DB와 달리 파일은 이미 쓰여진 데이터를 롤백할 수 없다는 문제가 있다. 
>이 문제를 해결하기 위해 FlatFileItemWriter는 데이터를 즉시 파일에 쓰지 않고 내부 버퍼에 일시적으로 저장해둔다.
![[Pasted image 20250925172936.png]]
- `FlatFileItemWriterBuilder`의 `transactional()` 메소드로 설정할 수 있으며, 기본값은 `true`이다.
- 이 후 청크 처리가 정상적으로 완료되어 트랜잭션이 커밋되려고 할 때, 즉 `beforeCommit()` 콜백이 호출될 때 비로소 버퍼의 데이터를 파일에 쓴다.

# `FlatFileItemWriter`구조
## 1단계: FieldExtractor(필드 추출)

![[Pasted image 20250923172411.png]]

```java
// FieldExtractor 내부
public interface FieldExtractor<T> { 
	Object[] extract(T item); 
}

---

// 동작 예시
public class DeathNoteFieldExtractor implements FieldExtractor<DeathNote> {
    @Override
    public Object[] extract(DeathNote deathNote) {
        return new Object[]{deathNote.getName(), deathNote.getCauseOfDeath()};
    }
}
```
- **`extract()`동작** 
	- 객체(`T`)에서 필드를 추출해서 `Object[]`로 반환
#### `FieldExtractor`구현체
> 스프링 배치에서는 파일에 쓸 도메인 객체의 타입에 따라 아래 두 구현체 중 하나를 자동으로 선택한다.
- **`BeanWrapperFieldExtractor`** 
	- Java Bean 객체로부터 필드를 추출하는 FieldExtractor로, 객체의 프로퍼티 이름들을 기반으로 getter 메서드를 호출하여 필드값을 추출한다.****
- **`RecordFieldExtractor`**
	- Java Record 객체에서 필드를 추출하는 FieldExtractor로, 레코드 컴포넌트의 accessor 메서드를 호출하여 값을 가져온다.
## **2단계: LineAggregator(문자열 결합)**
>내부적으로 `LineAggregator`의 `aggregate()` 메서드 안에서 `FieldExtractor`의 필드 추출 및 문자열 결합이 진행 된다.
![[Pasted image 20250923175428.png]]
 ```java
// LineAggregator 내부
public interface LineAggregator<T> {
    String aggregate(T item);
}

// 동작 예시
// FlatFileItemWriter.doWrite() -> write()에서 호출
public String doWrite(Chunk<? extends T> items) {
	StringBuilder lines = new StringBuilder();
	for (T item : items) { 
		lines.append(this.lineAggregator.aggregate(item))
		   .append(this.lineSeparator);
	}
	return lines.toString();
}
 ```
- LineAggregator 구현체를 사용할지는 FlatFileItemWriterBuilder의`delimited()`, `formatted()` 메서드를 사용하여 지정할 수 있다.
- `T`는 결합할 도메인 객체의 타입을 의미하며, `aggregate()` 메서드는 이 `T` 타입 객체를 받아 문자열로 결합하여 반환한다.
#### `LineAggregator` 구현체
- `DelimitedLineAggregator`
	- 구분자 기반 문자열로 변환
- `FormatterLineAggregator`
	- 커스텀 포맷 문자열로 변환
