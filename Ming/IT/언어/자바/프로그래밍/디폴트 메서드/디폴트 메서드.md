## 디폴트 메서드 도입 이유
- 자바 8에 처음 등장한 기능
- 인터페이스 내에서 선언되는 메서드이지만, 구현부를 가질 수 있음
- `default`키워드를 사용
- 추가된 이유: 이미 배포된 인터페이스에 새 메서드를 추가할 때 발생하는 **하위 호환성 문제**를 해결하기 위해

## 디폴트 메서드의 올바른 사용법
1. 하위 호환성을 위해 최소한으로 사용
	- 디폴트 메서드가 만들어진 목적에 알맞게 배포된 인터페이스에 새로운 메서드 추가가 필요하다면 고민해보자
	- 새 메서드가 필요한 상황이고, 구현 클래스가 많지 않다면 구현 클래스에 메서드를 추가하자
2. 인터페이스는 여전히 추상화의 역할
	- 계약의 역할에 충실한 것이 좋다.
	- 로직은 구현 클래스나 별도 클래스에 둘 것
3. 다중 상속(충돌) 문제
	- 여러 인터페이스에서 동일한 시그니처의 디폴트 메서드가 존재하면 충돌이 발생한다.
	- 이 경우 구현 클래스에서 메서드를 재정의 해야한다.
		1. 직접 구현 하거나
		2. `super`를 사용하여 사용할 인터페이스를 지정 해야한다.
4. 디폴트 메서드에 상태를 두지 않기
