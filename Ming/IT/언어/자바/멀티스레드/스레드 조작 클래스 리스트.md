### 1. 기본 기초 클래스 (The Basics)
가장 저수준에서 스레드를 정의하고 관리합니다.
- **`Thread`**: 스레드 실행의 본체.
- **`Runnable`**: 실행할 로직을 담는 인터페이스 (반환값 없음).
- **`Callable`**: 실행 후 결과값을 반환하거나 예외를 던질 수 있는 인터페이스.
- **`ThreadGroup`**: 스레드들을 묶어서 관리하는 집합.
- **`ThreadLocal`**: 스레드 단위의 로컬 변수를 저장하는 공간.
- **`InheritableThreadLocal`**: 부모 스레드의 값을 자식 스레드가 물려받는 ThreadLocal.
### 2. Executor 프레임워크 (스레드 풀 및 실행 제어)
스레드의 생성과 생명주기를 효율적으로 관리합니다.
- **`Executor`**: 작업을 실행하는 가장 단순한 인터페이스.
- **`ExecutorService`**: 작업을 종료하거나 결과를 추적하는 기능이 추가된 인터페이스.
- **`ScheduledExecutorService`**: 특정 시간 이후나 주기적으로 작업을 실행.
- **`Executors`**: 다양한 종류의 스레드 풀을 생성하는 팩토리 클래스.
- **`ThreadPoolExecutor`**: 가장 대표적인 스레드 풀 구현체.
- **`ScheduledThreadPoolExecutor`**: 타이머 기능을 갖춘 스레드 풀 구현체.
- **`Future`**: 비동기 작업의 결과를 나타내는 인터페이스.
- **`FutureTask`**: `Future`와 `Runnable`을 모두 구현한 실행 가능한 결과 객체.
- **`CompletionService` / `ExecutorCompletionService`**: 여러 비동기 작업 중 먼저 완료된 순서대로 결과를 처리할 때 사용.
- **`ScheduledExecutorService`**: 지연 및 주기적 실행을 위한 인터페이스.
- **`ScheduledThreadPoolExecutor`**: 스케줄링 기능이 강화된 스레드 풀.
- **`ScheduledFuture`**: 예약된 작업의 상태를 확인하거나 취소할 때 사용.
- **`Timer` / `TimerTask`**: 구형 스케줄링 도구.
### 3. Fork/Join 프레임워크 (분할 정복 병렬 처리)
큰 작업을 쪼개서 병렬로 처리할 때 사용합니다.
- **`ForkJoinPool`**: Fork/Join 작업을 수행하는 전용 스레드 풀 (Work-stealing 방식).
- **`ForkJoinTask`**: Fork/Join 풀에서 실행되는 작업의 기본 단위.
- **`RecursiveTask`**: 반환값이 있는 분할 정복 작업.
- **`RecursiveAction`**: 반환값이 없는 분할 정복 작업.
- **`CountedCompleter`**: 작업 완료 후의 후속 동작을 정의하는 태스크.
### 4. 동기화 도구 (Synchronizers)
스레드 간의 실행 순서나 흐름을 제어합니다.
- **`Semaphore`**: 허가(Permit)를 이용해 자원 접근 개수를 제한.
- **`CountDownLatch`**: 특정 개수의 작업이 끝날 때까지 대기.
- **`CyclicBarrier`**: 여러 스레드가 약속된 지점에 모두 모일 때까지 대기.
- **`Exchanger`**: 두 스레드가 서로 데이터를 맞교환.
- **`Phaser`**: `CountDownLatch`와 `CyclicBarrier`를 합친 것보다 더 유연한 단계별 동기화 도구.
### 5. 명시적 락 (Locks)
`synchronized` 키워드보다 정밀한 제어가 가능한 락입니다.
- **`Lock`**: 기본 락 인터페이스.
- **`ReentrantLock`**: 재진입이 가능한 표준 독점 락.
- **`ReadWriteLock`**: 읽기용 락과 쓰기용 락을 분리하여 관리.
- **`ReentrantReadWriteLock`**: `ReadWriteLock`의 구현체.
- **`Condition`**: 락과 연결되어 스레드 대기/깨움을 제어 (`wait/notify` 역할).
- **`StampedLock`**: (Java 8 추가) 낙관적 읽기(Optimistic Reading) 기능을 제공하는 고성능 락.
### 6. 원자적 변수 (Atomic Classes)
락 없이도 안전하게 값을 변경할 수 있게 해줍니다.
- **`AtomicInteger`, `AtomicLong`, `AtomicBoolean`, `AtomicReference`**: 기본 타입 및 참조형의 원자적 조작.
- **`AtomicIntegerArray`, `AtomicLongArray`, `AtomicReferenceArray`**: 배열 요소의 원자적 조작.
- **`AtomicReferenceFieldUpdater` 등**: 객체의 필드를 원자적으로 업데이트.
- **`LongAdder`, `DoubleAdder`**: (Java 8 추가) 쓰기 경합이 매우 심할 때 성능이 극대화된 카운터.
- **`LongAccumulator`, `DoubleAccumulator`**: (Java 8 추가) `LongAdder`의 일반화 버전으로 커스텀 연산 가능.
### 7. 스레드 안전한 컬렉션 (Concurrent Collections)
- **`ConcurrentHashMap`**: 멀티스레드 환경에 최적화된 Map.
- **`CopyOnWriteArrayList` / `CopyOnWriteArraySet`**: 읽기 작업이 압도적으로 많을 때 사용하는 컬렉션.
- **`ConcurrentSkipListMap` / `ConcurrentSkipListSet`**: 정렬된 상태를 유지하는 스레드 안전 컬렉션.
- **`BlockingQueue` 계열**: 스레드 간 데이터 교환용 (큐가 비거나 차면 대기).
    - `ArrayBlockingQueue`, `LinkedBlockingQueue`, `PriorityBlockingQueue`, `DelayQueue`, `SynchronousQueue`, `LinkedTransferQueue`.
- **`BlockingDeque` / `LinkedBlockingDeque`**: 양방향 끝에서 넣고 뺄 수 있는 블로킹 큐.
- **`ConcurrentLinkedQueue` / `ConcurrentLinkedDeque`**: 논블로킹 방식의 효율적인 큐.
### 8. 비동기 및 병렬 스트림 (Java 8 Specials)
- **`CompletableFuture`**: 비동기 프로그래밍의 핵심. 결과 조합, 콜백 등록 등이 가능.
- **`CompletionStage`**: `CompletableFuture`가 구현하는 비동기 단계 인터페이스.
- **`ParallelStream`**: 내부적으로 `ForkJoinPool`을 사용하여 데이터를 병렬 처리.
- **`Spliterator`**: 데이터를 병렬로 쪼개기 위한 반복자.
