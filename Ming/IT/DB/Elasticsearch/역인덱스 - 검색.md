## 역인덱스란?
-  필드 값을 단어마다 쪼개서 찾기 쉽게 정리해놓은 목록이다.
#### 검색
```json
// 검색 조회
GET /products/_search
{
  "query": {
    "match": {
      "name": "2026맥북"
    }
  }
}
```

#### 동작 방식
``` json
// POST /products/_doc로 저장
"name": "Apple 2025 맥북 에어 13 M4 10코어"
"name": "Apple 2024 에어팟 4세대"
"name": "Apple 2024 아이패드 mini A17 Pro"
```
1. 단어 단위로 자르기
	- (Apple, 2025, 맥북, 에어, 13, M4, 10코어)
	- (Apple,2024,에어팟,4세대)
	- (Apple,2024,아이패드,mini,A17,Pro)
2. 역인덱스로 저장
	- 필드값에서 추출돼 역인덱스에 저장된 단어를 보고 **토큰**이라고 부른다.
![[Pasted image 20260301175750.png]]
3. 검색을 할 경우 역인덱스 활용
	- "Apple 2024 아이패드"라고 검색하면 역인덱스를 활용해 일치하는 단어가 많은 다큐먼트를 우선적으로 조회
4. score 계산로직
	1. Tern Freqyebct: 문서 내에서 검색어가 얼마나 자주 등장하냐
	2. Inverse Document Frequency: 검색어가 전체 문서중 얼마나 희귀하냐
	3. Field Length Normalization: 문서가 짧을수록 관련성이 높다고 판단.
 이와 같은 Elasticsearch의 자체적인 로직으로 score를 매겨 score가 높은 순서로 도큐먼트를 조회한다.
> 이런 작동 구조는 데이터 타입의 `text`타입에 한해서만 적용된다.
