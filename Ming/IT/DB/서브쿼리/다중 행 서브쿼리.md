- 서브쿼리의 결과가 여러 행으로 반환되는 것이 당연할 때 사용하는 것이 다중 행 서브쿼리이다.
- 목록을 다루는 연산자 : `IN`, `ANY`, `ALL`이 있다.


## `ANY`, `ALL`연산자
`ANY`와 `ALL`은 주로 `>`, `<`와 같은 비교 연산자와 함께 사용되어, 서브쿼리가 반환한 여러 값들과 비교하는 역할을 한다.
- `> ANY (서브쿼리)`
	- 서브쿼리가 반환한 여러 결과값 중 어느 하나보다 크면 참이다.
	- 즉, 최소값보다 크면 참이다.
- `> ALL (서브쿼리)`
	- 서브쿼리가 반환한 여러 결과값 모두보다 커야만 참이다.
	- 즉, 최대값보다 커야 참이 된다.
- `< ANY (서브쿼리)`
	- 최대값보다 작으면 참이다.
- `< ALL (서브쿼리)`
	- 최소값보다 작으면참이다
- `\= ANY (서브쿼리)`
	- IN과 완전히 동일한 의미이다.

#### `> ANY (서브쿼리)`
```sql
SELECT NAME, PRICE
FROM PRODUCTS
WHERE PRICE > ANY (
	SELECT PRICE FROM PRODUCTS WHERE CATEGORY='전자기기'
);
```

>이 쿼리는 `price > 75000` OR `price > 120000` OR `price > 350000` OR `price > 280000` 와 같이 동작한다.


## 실무
사실 `ANY`와 `ALL`은 `MIN`과 `MAX`를 사용한 코드로 대체될 수 있어서 `IN`보다 사용 빈도는 낮은 편이다.
