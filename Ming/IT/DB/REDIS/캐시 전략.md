## 캐시란
원본 저장소보다 빠르게 가져올 수 있는 임시 데이터 저장소를 의미

## 전략
##### Cache Aside 전략
1. Cache Hit: 캐시에 데이터가 있을 경우
2. Cache Miss: 캐시에 데이터가 없을 경우
##### Write Around 전략
- 쓰기 작업을 캐시에는 반영하지 않고  DB에만 반영하는 방식을 뜻한다.
##### Write Back 전략
- 클라이언트의 요청이 들어오면 먼저 Redis에 데이터를 쌓아두고 클라이언트에게 먼저 응답을 반환하고나서 Redis에 쌓인 데이터를 스케줄러가 주기적으로 DB에 한꺼번에 저장하는 방식을 말한다.

## 한계점과 해결방법
- Cache Aside와 Write Around 전략을 같이 썼을 때 한계점중 하나는 캐시된 데이터와 DB데이터가 일치하지 않을 수 있다는 점이다.
- 자주 조회되고 잘 변하지 않고 실시간으로 일치하지 않아도 되는 데이터에 TTL을 설정하여 사용하면 된다.
