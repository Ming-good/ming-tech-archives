## List
순서대로 데이터를 저장할 수 있는 자료구조
- `RPUSH KEY VALUE ...` :  오른쪽에서 부터 데이터를 넣어준다.
- `LPUSH KEY VALUE ...` : 왼쪽에서 부터 데이터를 넣어준다.
- `LRANGE KEY start end` :   start ~ end까지 데이터를 조회한다.
- `LLEN KEY`: List Size 
- `LPOP my_list`: 좌측으로 부터 데이터를 꺼낸다
- `RPOP my_list`: 우측으로 부터 데이터를 꺼낸다.
## SET, GET, DEL
- `SET KEY VALUE`: SET자료구조에 저장한다.
- `SET KEY VALUE NX`: 중복된 데이터는 저장하지 않는다
- `DEL KEY`: 해당 키를 삭제한다.

## ZSET
> Redis에서 Sorted Set(ZSET)은 score(점수)를 기준으로 자동 정렬되는 중복없는 자료구조 이다. 
> 이런 특성 때문에 실제 서비스에서 랭킹 시스템, 실시간 인기 데이터 관리처럼 순서와 가중치가 중요한 경우 주로 활용된다.
- `ZADD`
```bash
# user_scores라는 키에 "alice"는 100점, "bob"은 200점 저장
ZADD user_scores 100 "alice"
ZADD user_scores 200 "bob"
```

- `ZRANGE (key) (start num)`
```bash
# 0등부터 10등까지 데이터 가져오기 (이름만)
ZRANGE user_scores 0 10
# 점수까지 같이 보고 싶을 때
ZRANGE user_scores 0 -1 WITHSCORES
# 역순(높은 점수부터)으로 가져오기
ZRANGE user_scores 0 -1 REV WITHSCORES
```

- `ZREMRANGEBYSCORE (key) (start num) (end num)`
```bash
# 점수가 0점부터 150점 사이인 데이터를 싹 삭제
ZREMRANGEBYSCORE user_scores 0 150
```

- `ZSCORE (key) (target)`   
- `ZRANK (key) (target)`
```bash
# "alice"의 현재 점수 확인
ZSCORE user_scores "alice"

# "bob"이 몇 등인지 확인 (0부터 시작)
ZRANK user_scores "bob"
```

- `ZCARD (key)`
- `ZCOUNT (key) (start num) (end num)`
```bash
# 전체 데이터가 몇 개인지
ZCARD user_scores

# 100점 ~ 200점 사이인 데이터만 몇 개인지
ZCOUNT user_scores 100 200
```
